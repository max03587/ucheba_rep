# Менеджеры пакетов Python: полный обзор

## 1. PyPI — центральный репозиторий разработчиков python

**PyPI** (Python Package Index) — это **реестр пакетов**, центральное хранилище, расположенное на `pypi.org`. На нём находится более 500 000 проектов. PyPI **не устанавливает** пакеты сам — для этого нужен клиент.

|Характеристика|Значение|
|---|---|
|URL|[https://pypi.org](https://pypi.org/)|
|Формат пакетов|`.whl` (wheel), `.tar.gz` (sdist)|
|API|Simple API, JSON API|
|Кто управляет|Python Software Foundation (PSF)|

> PyPI — это **склад**. Менеджеры забирают оттуда пакеты, но сами не являются PyPI.

---

## 2. pip — классический стандарт

**pip** (Pip Installs Packages) — официальный пакетный менеджер Python, поставляемый вместе с Python 3.4+.

### Плюсы

- Поставляется "из коробки" с Python
- Простота: `pip install requests`
- Гибкая работа с разными индексами (`--index-url`, `--extra-index-url`)
- Поддержка requirements.txt, editable installs, wheel

### Минусы

- **Нет разрешения зависимостей** (dependency resolver появился только в pip 20.3+ и он медленный)
- Нет изоляции окружений (нужен venv отдельно)
- Нет управления проектом (не умеет создавать метаданные, упаковывать пакеты)


---

## 3. Conda — не только Python

**Conda** — пакетный менеджер из экосистемы Anaconda, который умеет устанавливать не только Python-пакеты, но и системные библиотеки (C/C++, R, инструментарии).

### Ключевые отличия от pip

|Характеристика|pip|Conda|
|---|---|---|
|Среда установки|Только Python|Любые бинарники|
|Разрешение зависимостей|SAT-решатель (медленный)|SAT-решатель (быстрее)|
|Изоляция|Требует venv|Встроена в conda env|
|Источники|PyPI + др. индексы|conda-forge, defaults, bioconda|
|Бинарные пакеты|wheel (.whl)|.conda / .tar.bz2|

### Плюсы

- Отличное разрешение зависимостей: Conda умеет подбирать версии Python и библиотек так, чтобы всё совместилось
- Предустановленные бинарники для Data Science (numpy, scipy, tensorflow — с оптимизациями под процессор)
- Изолированные окружения встроены (`conda create -n myenv python=3.11`)
- Работает на Windows, Linux, macOS

### Минусы

- **Большой размер** (Anaconda — 3+ ГБ, Miniconda — 500+ МБ)
- Медленный старт (решатель зависимостей грузит все метаданные)
- Не все пакеты есть на conda-forge (приходится комбинировать с pip)
- Лицензионные ограничения для коммерческого использования (Anaconda репозиторий)

## 4. uv — сверхбыстрый современный преемник pip

**uv** — пакетный менеджер на Rust, разработанный Astral (создатели Ruff). Он позиционируется как **замена pip** с полной совместимостью, но на порядок быстрее.

### Скорость (бенчмарки Astral)

|Операция|pip (сек)|uv (сек)|
|---|---|---|
|Установка Flask|3.2|0.4|
|Установка 100 пакетов|45|2.8|
|Разрешение зависимостей|8.1|0.6|

### Возможности

- **Полная совместимость с pip**: `uv pip install flask` — те же флаги
- **Быстрый resolve** — кэширование и параллельные HTTP-запросы
- **Встроенный venv**: `uv venv` создаёт виртуальное окружение мгновенно
- **Pip sync** — эквивалент `pip install -r requirements.txt`, но быстрее в 10–50 раз
- **Lock-файлы**: `uv.lock` для воспроизводимых установок
- **Управление версиями Python**: `uv python install 3.12`

### Плюсы

- Огромная скорость
- Совместимость с существующими проектами и requirements.txt
- Небольшой бинарник (один файл, 10–15 МБ)
- Открытый исходный код (MIT)

### Минусы

- **Молодой проект** (первый релиз — февраль 2024, текущая версия ~0.6.x)
- Нет встроенного управления проектом (версионирование, публикация на PyPI)
- Некоторые экзотические флаги pip ещё не реализованы

## 5. Poetry — управление проектом и зависимостями

**Poetry** — инструмент для **управления Python-проектами**: зависимости, упаковка, публикация на PyPI.

### Ключевая идея

Poetry объединяет:

- `pip` (установка)
- `venv` (виртуальные окружения)
- `setup.py` / `setup.cfg` (метаданные пакета)
- `requirements.txt` (зависимости)

Всё это в одном инструменте с единым `pyproject.toml`.


### Плюсы

- **Единый источник истины** — `pyproject.toml` вместо нескольких файлов
- **Чёткое разделение** зависимостей и dev-зависимостей
- **Lock-файл** (`poetry.lock`) для воспроизводимости
- **Встроенная публикация**: `poetry publish`
- **Автоматическое создание виртуального окружения**
- **Группы зависимостей**: `[tool.poetry.group.docs.dependencies]`

### Минусы

- **Медленный resolve** (особенно на старых версиях, сейчас улучшили)
- **Не во всех проектах нужен** — если вы работаете над библиотекой, Poetry оправдан; если просто скрипт — оверхед
- **Неудобно с pip-стилем** — привыкшие к `pip freeze` страдают от отсутствия прямой совместимости с requirements.txt
- **Версия Python фиксируется в toml** — при смене версии надо править файл вручную

## 6. PDM — современный менеджер с PEP 582

**PDM** (Python Development Master) — ещё один современный менеджер, следующий стандарту **PEP 582** (локальная установка без виртуального окружения).

### Отличительная особенность

PDM поддерживает **PEP 582** — установку пакетов в `__pypackages__` в корне проекта, **без создания виртуального окружения** и активации.

```
my-project/
├── __pypackages__/     # пакеты устанавливаются сюда
│   └── 3.11/
│       └── lib/
├── src/
├── pyproject.toml
└── pdm.lock
```

### Дополнительные возможности

- **Поддержка нескольких форматов lock-файлов**: PDM lock, pip-style requirements.txt
- **Поддержка PEP 517/518/621** — полный стек современных стандартов
- **Встроенная публикация**
- **Resolve через PyPI и Conda** (экспериментально)
- **Кэширование** — ускоряет повторные установки
- **Плагины** — расширяемость через плагинную систему

### Плюсы

- PEP 582 — работа без активации окружения, `pdm run python script.py`
- Быстрый resolve (на основе `pylock`)
- Хорошая документация
- Совместимость с `pyproject.toml`

### Минусы

- PEP 582 пока не стал мейнстримом (поддерживается не всеми инструментами)
- Меньше сообщества и готовых решений, чем у Poetry
- Иногда проблемы с Windows и сложными C-расширениями

## Практические рекомендации

### Для быстрых экспериментов и скриптов

```
pip с venv
или uv (если установлен) — на порядок быстрее
```

### Для Data Science проектов

```
Conda — бинарные numpy/scipy с оптимизациями
+ pip для всего остального
```

### Для библиотек и публикуемых пакетов

```
Poetry — встроенная публикация, полный цикл разработки
```

### Для проектов, где важна скорость CI/CD

```
uv — на порядок сокращает время установки в CI
```

### Если хочется "без venv"

```
PDM с PEP 582 — не нужно активировать окружение
```

## Краткие итоги

|Инструмент|Лучше всего подходит для|
|---|---|
|**PyPI**|Хранилище пакетов, не инструмент|
|**pip**|Базовые установки, совместимость, наследие|
|**Conda**|Data Science, системные библиотеки, ML|

| **uv** | Скорость, замена pip, современные CI/CD | | **Poetry** | Разработка и публикация библиотек | | **PDM** | Современные проекты, PEP 582, гибкость |

Сегодняшний тренд — **uv** как самый быстрый установщик и **Poetry** как самый удобный для управления проектами. Conda остаётся стандартом в Data Science.
