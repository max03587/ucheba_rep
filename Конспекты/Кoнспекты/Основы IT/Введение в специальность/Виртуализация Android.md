**Виртуализация Android** — это технология, которая позволяет запускать несколько операционных систем или изолированных сред на одном физическом устройстве. Это достигается с помощью **виртуальных машин (VM)** — программных конструкций, которые эмулируют физическую вычислительную среду. Каждая VM работает как отдельный компьютер с собственными операционной системой и приложениями.

На уровне системы. Например:

Kernel-based Virtual Machine (KVM) — действует как гипервизор, облегчает прямое взаимодействие гостевых ОС с оборудованием.
Virtio Framework — устанавливает оптимизированный канал связи между гостевой и хостовой операционными системами.

### Различия
Виртуализация для Android и систем для ЭВМ отличается тем, что в Android виртуализация — это технология, встроенная в операционную систему, а в системах для ЭВМ — это технология, позволяющая создавать виртуальные версии физических ресурсов. 

### Виртуализация для Android
Android Virtualization Framework (AVF) — это технология, которая обеспечивает изоляцию виртуальных машин (ВМ) в рамках операционной системы Android. Некоторые особенности: 

- Выбор уровня изоляции:
Односторонняя изоляция — Android (хост) может контролировать и проверять содержимое ВМ.
Двусторонняя изоляция (изолированная ВМ) — Android (хост) и ВМ (гость) полностью изолированы друг от друга.
- Портативность — виртуальные машины и приложения, запущенные в них, более портативны, чем доверенные апплеты. Например, виртуальная машина на основе Linux с приложением на Linux будет работать на всех устройствах, поддерживающих AVF.
- Поддержка только на устройствах ARM64 — AVF поддерживается только на этой платформе.

### Системы для ЭВМ
Виртуализация для вычислительных систем (ЭВМ) — это технология, которая позволяет создавать виртуальные версии физических ресурсов: серверов, хранилищ, сетей и даже операционных систем. Некоторые особенности: 

- Использование гипервизора — программы, которая управляет виртуальными машинами и распределяет ресурсы между ними.
- Возможность запускать на одном сервере несколько виртуальных машин, каждая из которых использует часть ресурсов физического сервера.
- Гибкость — по мере появления новых требований аппаратные ресурсы гибко перераспределяются между виртуальными машинами.
